# "轨迹" MVP 产品设计文档

## 一句话定义

一个"有你记忆的城市地图"——点亮去过的城市，每个城市有你的照片、AI 专属印章和到访日期。

## 核心痛点

高德地图的城市点亮功能只展示城市的公共信息，没有任何与用户个人相关的记忆。用户点亮了一个城市，点开后却看不到自己的照片、自己的故事。"轨迹"要解决的是：让每一个点亮的城市都承载属于你的记忆。

## 目标用户

高频旅行者（平均每月 1-2 次旅行，短途或长途均有），希望用一种有仪式感的方式记录每一次旅程，积累城市印章集合，并能生成好看的分享图发朋友圈。不追求覆盖所有旅行用户，聚焦服务"经常在路上"的人群。

## MVP 范围

### 做什么

- 中国/世界地图，去过的城市高亮显示
- 上传照片自动识别城市（读取 EXIF GPS + 逆地理编码）
- 每个城市生成一枚 AI 艺术印章（风格可选：国潮/极简/复古/赛博朋克）
- 城市记忆卡片：一张代表照片 + AI 印章 + 到访日期
- 护照集邮页面（翻页浏览所有印章）
- 旅行统计数据（总城市数、总公里数、最远到达、覆盖省份比例等）
- 生成分享海报（竖版 9:16，含统计数据，适合朋友圈）
- 旅行中支持手动打卡（获取当前定位 + 拍照）
- 手机号验证码登录

### 不做（MVP 阶段）

- 多人/双人/家庭功能
- 完整旅行故事 HTML 页面
- AI 攻略生成
- 实体打印下单
- 用户社区/评论/关注
- 微信小程序端（Web 先行）

## 用户流程

### 流程 A：旅行后补录（主流程）

1. 用户点击"上传照片"
2. 选择多张旅行照片
3. 前端读取每张照片的 EXIF（GPS 坐标 + 拍摄日期）
4. 调用高德逆地理编码 API，将坐标转为城市名
5. 按城市分组展示："检测到你去过 厦门(5.10-5.12, 47张)、泉州(5.13, 12张)"
6. 用户为每个城市选一张代表照片
7. 用户选择印章风格
8. 系统调用 AI 生成城市印章（约 10 秒）
9. 城市在地图上点亮，记忆卡片生成完毕

### 流程 B：旅行中现场打卡

1. 用户点击"现场打卡"
2. 系统获取当前 GPS 定位，识别城市
3. 用户拍照或从相册选一张
4. 选择印章风格 → AI 生成印章
5. 城市点亮

### 流程 C：分享

1. 用户在地图上点击已点亮城市
2. 查看城市记忆卡片（照片 + 印章 + 日期）
3. 点击"分享" → 生成竖版海报图
4. 海报包含：地图缩略图（高亮该城市）+ 照片 + 印章 + 日期 + "我的第X个城市"
5. 长按保存 / 分享到微信

## 页面结构

### 整体导航：底部三个 Tab

```
📍 地图    |    📖 护照    |    🎞️ 记忆
```

- **地图 Tab**：城市点亮地图，打卡入口
- **护照 Tab**：印章集邮页面（按城市，轻量收集感）
- **记忆 Tab**：旅程故事列表（按旅程，丰富沉浸感）

### 核心概念区分

| | 印章（护照 Tab） | 记忆（记忆 Tab） |
|---|---|---|
| 粒度 | 每个**城市**一个 | 每次**旅程**一个 |
| 形态 | 一张印章图片 | 完整 HTML 页面（多照片+文案+路线） |
| 感觉 | 轻量、快速、收集感 | 沉浸、丰富、回味感 |
| 类比 | 护照上的签证章 | Apple 照片的"回忆" |

### 页面 1：地图 Tab（首页）

- 全屏地图（高德 JS API）
- 去过的城市用亮色圆点标记
- 右上角统计数字："已点亮 X 个城市"
- 底部两个按钮："上传照片"（旅行后）/ "现场打卡"（旅行中）
- 点击已点亮城市 → 弹出城市卡片（代表照片+印章+日期）

### 页面 2：护照 Tab（印章集邮）

- 翻页效果（像真实护照一样左右滑动）
- 每页 2-4 个印章，按时间顺序排列
- 顶部统计："已收集 X/337 城"（337 = 中国地级市数量）
- 长按某个印章 → 查看该城市的记忆卡片
- 可切换视图：护照翻页模式 / 网格九宫格模式

### 页面 3：记忆 Tab（旅程故事列表）

- 按时间倒序展示所有旅程
- 每条旅程卡片：封面照片 + 旅程名称 + 日期 + 途经城市标签
- 点击进入旅程详情（HTML 故事页面）
- MVP 阶段：此 Tab 显示"即将上线"占位
- v1.0 阶段：简版旅程页面（照片时间轴 + 地图路线）
- v2.0 阶段：完整版（AI 文案 + 背景音乐 + 翻页动画）

### 页面 4：上传识别页

- 照片选择区域
- 识别结果分组展示（按城市，自动归为一次旅程）
- 每个城市可选代表照片
- 印章风格选择（4 种风格的预览缩略图）
- "生成印章"按钮 + 加载动画

### 页面 5：城市记忆卡片（弹窗/抽屉）

- 代表照片（大图，圆角）
- AI 印章（居中展示）
- 城市名 + 到访日期
- "分享"按钮
- "查看旅程"按钮（跳转到该城市所属旅程的记忆页面）

### 页面 6：分享海报生成页

- 海报预览（竖版 9:16）
- "保存到相册"按钮
- "分享给朋友"按钮

### 页面 7：个人设置

- 头像 + 昵称
- 设置：修改昵称、退出登录

### 统计数据系统

在护照 Tab 顶部 + 分享海报中展示，包含以下维度：

- 已点亮城市数 / 337（中国地级市总数）
- 覆盖省份数 / 34
- 累计旅行公里数（基于城市间直线距离估算）
- 最远到达的城市（距离家乡最远的打卡点）
- 最北/最南/最东/最西打卡点
- 今年新增城市数
- 连续打卡月数（连续 X 个月都有新城市）

统计数据同时用于分享海报，增强"炫耀价值"——如"我已走过中国 15% 的城市，最远到达拉萨（距家乡 3,200km）"。

## 数据结构

### User（用户）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| phone | string | 手机号 |
| nickname | string | 昵称 |
| avatar_url | string | 头像 URL |
| hometown_city | string | 家乡城市（首次引导设置） |
| created_at | timestamp | 注册时间 |

### CityFootprint（城市足迹）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| city_name | string | 城市名（如"厦门市"） |
| province | string | 省份（如"福建省"） |
| country | string | 国家（如"中国"） |
| latitude | float | 城市中心纬度 |
| longitude | float | 城市中心经度 |
| first_visit_date | date | 首次到访日期 |
| latest_visit_date | date | 最近到访日期 |
| cover_photo_url | string | 代表照片 URL |
| stamp_image_url | string | AI 印章图片 URL |
| stamp_style | enum | 印章风格：guochao/minimal/retro/cyberpunk |
| created_at | timestamp | 记录创建时间 |

### Trip（旅程）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| title | string | 旅程名称（如"厦门泉州之旅"，可 AI 生成） |
| start_date | date | 旅程开始日期 |
| end_date | date | 旅程结束日期 |
| cover_photo_url | string | 旅程封面照片 |
| memory_html_url | string | 记忆 HTML 页面 URL（v1.0+） |
| status | enum | draft / generated |
| created_at | timestamp | 创建时间 |

一次旅程可以关联多个城市（厦门+泉州），印章是每个城市一个，记忆故事是整趟旅程一个。

### Photo（照片）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| city_footprint_id | UUID | 关联城市足迹 |
| trip_id | UUID | 关联旅程 |
| photo_url | string | 照片存储 URL |
| original_lat | float | 原始 GPS 纬度 |
| original_lng | float | 原始 GPS 经度 |
| taken_at | timestamp | 拍摄日期（EXIF） |
| is_cover | boolean | 是否为该城市代表照片 |
| uploaded_at | timestamp | 上传时间 |

## 技术架构

### 技术栈

| 层级 | 选择 | 理由 |
|------|------|------|
| 前端框架 | Next.js 14 (React) + Tailwind CSS | AI 辅助开发效率最高，Tailwind 免写 CSS |
| 后端/数据库 | Supabase | 一站式：PostgreSQL + Auth + Storage + Edge Functions |
| 地图 | 高德地图 JS API v2 | 中国地图体验好，逆地理编码 API 免费额度大 |
| EXIF 解析 | exifr（前端 JS 库） | 浏览器内直接读取照片 GPS/日期 |
| AI 印章生成 | 通义万相 API（主）/ Replicate FLUX（备） | 国内稳定，中文城市理解好 |
| 图片压缩 | browser-image-compression | 上传前压缩到 1MB 以内 |
| 海报生成 | html2canvas | 前端将 DOM 渲染为图片 |
| 部署 | Vercel | 免费，零配置，自动 HTTPS |

### 架构图

```
浏览器（微信内/普通浏览器）
    │
    ├── Next.js 前端（Vercel 部署）
    │     ├── 地图页面（高德 JS API）
    │     ├── 照片上传 + EXIF 解析（exifr）
    │     ├── 城市卡片展示
    │     └── 海报生成（html2canvas）
    │
    ├── Supabase
    │     ├── Auth（手机号+验证码登录）
    │     ├── PostgreSQL（城市足迹、照片数据）
    │     ├── Storage（照片文件、印章图片）
    │     └── Edge Functions（调用 AI API 生成印章）
    │
    └── 外部 API
          ├── 高德逆地理编码（坐标→城市）
          └── 通义万相（文生图，生成印章）
```

### AI 印章生成策略

- Top 50 热门城市：预先设计高质量印章模板作为兜底
- 其他城市：调用 AI 实时生成
- Prompt 模板示例：`"一枚圆形旅行印章，{city_name}主题，包含{landmarks}地标元素，{style}风格，白色背景，适合收藏"`
- 生成失败时自动 fallback 到通用模板（城市名+日期的简约印章）

## 关键设计决策

### 1. 冷启动引导

首次打开时引导用户："选择你的家乡，点亮第一个城市"。家乡不需要照片验证，确保用户一进来地图就不是空的。

### 2. 无 GPS 照片处理

上传照片时如果检测不到 GPS 信息：
- 弹出提示："这张照片没有定位信息"
- 提供手动选择城市的选项
- 不默默丢弃，给用户补充信息的机会

### 3. 同一城市多次到访

同一城市只有一条足迹记录。多次到访时：
- 更新 `latest_visit_date`
- 新照片追加到该城市的照片集合
- 用户可以随时更换代表照片
- 印章保留首次生成的（除非用户主动重新生成）

### 4. 照片压缩策略

- 前端上传前自动压缩至 1MB 以内（保留 EXIF）
- 生成缩略图（200x200）用于列表展示
- 原图用于卡片大图和海报

### 5. 登录方式

- MVP 使用手机号 + 短信验证码（Supabase Auth 原生支持）
- 未来小程序端接入微信 OAuth
- 两端通过手机号关联同一账户

## 成功指标

MVP 阶段（前 2 个月）的核心验证指标：

- **自己是否愿意用**：你自己下一次旅行后，是否愿意用"轨迹"记录并分享
- **分享率**：生成海报后，是否真的发了朋友圈/发给了朋友
- **朋友反馈**：看到你分享的人，是否问"这是什么 App"

如果以上三个答案都是"是"，说明核心体验成立，值得继续投入。
